# Multi-Plant Supply Chain Optimization

Linear programming model in GAMS for optimizing a multi-plant electric scooter supply chain under tariffs, supplier constraints, service levels, and CO₂ emission limits.

## Project Overview
This project develops a linear programming model for a multi-plant electric scooter supply chain. The model determines optimal production, sourcing, shipment, inventory, and shortage decisions while minimizing total cost under tariff and environmental policy constraints.

## Problem Statement
The supply chain includes:
- Multiple production plants
- Multiple suppliers
- Multiple markets
- Component procurement decisions
- Tariff impacts
- CO₂ emission considerations
- Service level requirements

The goal is to find the most cost-effective and feasible production and distribution plan under different business scenarios.

## Tools Used
- **GAMS** for optimization modeling
- **Excel** for experimentation and result tracking
- **Linear Programming** for decision modeling

## Files in This Repository
- **FINALPROJECTCODE.gms** – Main GAMS optimization model
- **FINALPROJECTCODE.gms.log** – GAMS execution log file
- **FINALPROJECTCODE.lst** – Output listing file generated by GAMS
- **FINALPROJECTCODE.lxi** – GAMS solution-related file
- **EXPERIMENTATION.xlsx** – Scenario analysis, experiment results, or supporting calculations
- **Multi-Plant Production and Supply Chain Optimisation Under Tariff and Environmental Policies** – Final project report/documentation
- **README.md** – Project overview and repository guide

## Model Features
- Multi-plant production planning
- Supplier allocation
- Market-wise shipment decisions
- Capacity constraints
- Component availability constraints
- Inventory balance
- Shortage penalty modeling
- Tariff-based cost adjustments
- CO₂ emission limit considerations

## Scenarios Analyzed
- Base case optimization
- Tariff increase analysis
- Supplier disruption analysis
- Service level comparison
- Policy sensitivity analysis

## How to Run
1. Open `FINALPROJECTCODE.gms` in GAMS.
2. Run the model.
3. Check output in:
   - `FINALPROJECTCODE.lst`
   - `FINALPROJECTCODE.gms.log`
4. Review experiment summaries in `EXPERIMENTATION.xlsx`.
